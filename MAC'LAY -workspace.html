// Deployment Guide

Deploy MAC'LAY

Full deployment instructions for Vercel (frontend) and Railway (backend). Get your own instance live in minutes.

▲ Vercel — Frontend

Deploy the frontend (this HTML file) to Vercel for global CDN delivery and automatic HTTPS.

1
Install Vercel CLI: npm i -g vercel
2
In your project folder run: vercel login
3
Deploy with: vercel --prod
4
Add custom domain in Vercel dashboard → Settings → Domains → maclay.com
5
Set env var: ANTHROPIC_API_KEY in Vercel project settings

🚂 Railway — Backend API

Deploy the Node.js/Python backend on Railway for Judge0 code execution and user auth.

1
Go to railway.app → New Project → Deploy from GitHub
2
Add PostgreSQL plugin in Railway dashboard (free included)
3
Set env vars: ANTHROPIC_API_KEY, JUDGE0_API_KEY, JWT_SECRET
4
Railway auto-deploys on every GitHub push
5
Point frontend to Railway API: update BACKEND_URL in your env

⚡ Judge0 — Code Execution

Integrate Judge0 for safe, sandboxed multi-language code execution.

1
Get API key at judge0.com/pricing (2,500 free execs/month)
2
Add to Railway env: JUDGE0_API_KEY=your_key
3
Backend route: POST /api/execute proxies to Judge0
4
Supported: Python, Java, C, C++, PHP, Ruby, Go, Bash
5
Set timeout: 10s max per execution for safety

🔐 Auth — User Accounts

Set up Google OAuth and email/password authentication for user accounts.

1
Create Google OAuth app at console.cloud.google.com
2
Set env: GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET
3
Install: npm install passport passport-google-oauth20 bcryptjs jsonwebtoken
4
PostgreSQL table: users(id, email, name, provider, created_at)
5
JWT tokens stored in httpOnly cookies for security
